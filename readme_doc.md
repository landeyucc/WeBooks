## 必要文档块
| 模块标题                | 作用说明                                                                 |
|-------------------------|--------------------------------------------------------------------------|
| 1. 程序概述（Program Overview） | 核心定位：让读者 3 分钟了解程序核心。需包含：程序用途、解决的问题、核心特性、适用场景、不适用场景。 |
| 2. 快速上手（Quick Start）     | 核心定位：降低使用门槛，提供「最小化启动流程」。需包含：环境依赖、安装步骤、一键启动命令、简单验证方法（如运行后输出什么代表成功）。 |
| 3. 核心功能（Core Features）   | 核心定位：明确程序的核心能力边界。需包含：主要功能清单、每个功能的核心作用（无需展开操作，仅说明“能做什么”）。 |
| 4. 使用说明（Usage Guide）     | 核心定位：详细指导“如何用”。需包含：基础操作流程、关键参数说明（必填/可选、默认值、示例）、常用命令/API 示例、典型使用场景演示（如“如何实现 XX 需求”）。 |
| 5. 注意事项（Notes & Warnings） | 核心定位：规避用户踩坑。需包含：已知限制（如并发上限、数据格式要求）、常见错误及解决办法、禁忌操作（如不可修改的配置、可能导致崩溃的场景）。 |

## 可选文档块（如果存在或需要）
| 模块标题                          | 适用场景                                                                 | 作用说明                                                                 |
|-----------------------------------|--------------------------------------------------------------------------|--------------------------------------------------------------------------|
| 6. 详细配置（Configuration Details） | 需自定义配置的程序（如服务端、工具类）                                   | 包含：配置文件路径、所有配置项说明（键名、类型、默认值、用途）、配置示例。 |
| 7. 接口文档（API Documentation）   | 提供接口的程序（如后端服务、SDK、库）                                     | 包含：接口列表（REST/函数/方法）、请求参数（字段、类型、必填项）、响应格式、错误码说明、接口示例（请求+响应）。 |
| 8. 部署与运维（Deployment & Ops） | 需部署运行的程序（如后端服务、应用）                                       | 包含：部署环境要求、部署步骤（单机/集群）、启动/停止/重启命令、监控指标、日志位置、备份/恢复方案。 |
| 9. 开发指南（Development Guide）  | 开源程序、团队协作开发的程序                                             | 包含：开发环境搭建、代码目录结构、编码规范、构建/打包命令、提交规范、本地调试方法。 |
| 10. 依赖说明（Dependencies）       | 依赖第三方库/工具的程序                                                   | 包含：依赖清单（名称、版本号）、依赖安装方式（如 npm/yum/pip 命令）、依赖冲突解决。 |
| 11. 版本历史（Changelog）          | 持续迭代的程序                                                           | 按版本号（语义化版本）排列，记录每个版本的新增功能、修复的问题、不兼容变更。 |
| 12. 常见问题（FAQ）                | 使用者易产生疑问的程序（如复杂工具、高频使用的服务）                       | 整理用户高频咨询的问题及解决方案（如“启动失败怎么办？”“参数报错如何排查？”）。 |
| 13. 贡献指南（Contributing）       | 开源程序、接受外部贡献的程序                                             | 包含：贡献流程（Fork -> PR -> 审核）、代码提交规范、Issue 提交规范、 reviewers 信息。 |
| 14. 许可证（License）              | 开源程序、有版权要求的程序                                               | 说明程序的授权协议（如 MIT、Apache 2.0）、版权归属、使用限制。             |
| 15. 联系方式（Contact & Support）  | 需提供技术支持的程序（如商业软件、企业内部服务）                           | 包含：维护者信息、反馈渠道（Issue 地址、邮箱、群聊）、技术支持范围。       |
